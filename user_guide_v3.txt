   9. OPEN COUNT  Currently <5 swing positions open?

  Note: Check 1 (earnings) is FIRST so you don't waste analysis
  time on names that will fail anyway.


───────────────────────────────────────────────────────────────
SETUP DEFINITIONS — SWING ALPHA
───────────────────────────────────────────────────────────────

  PULLBACK (primary, 70% of trades) — needs ALL 7:
    1. Earnings runway clear (>10 trading days out)
    2. Confirmed uptrend (above 50-SMA, 50 > 200)
    3. Pullback structure: 3-8% deep, 3-15 days long,
       retrace to 21-EMA / 50-EMA / proven support
    4. Volume LOWER on pullback than prior advance
    5. Confirmation candle (hammer/engulfing/strong bull/inside)
    6. Entry: BUY STOP at $0.05 above confirmation candle high
    7. Stop: MIN of swing low or 1.5×ATR (whichever closer)

  BREAKOUT (secondary, 30% of trades) — needs ALL 6:
    1. Earnings runway clear (>10 trading days out)
    2. Consolidation base ≥4 weeks, <8% range
    3. Clear horizontal resistance with 2+ rejections
    4. Volume ≥150% of 20-day avg on breakout day
    5. Daily CLOSE above resistance (not just intraday)
    6. Next resistance ≥5% above breakout

  ENTRY for breakouts: BUY STOP at $0.10 above breakout day high
  STOP for breakouts: $0.10 below broken resistance

TROUBLESHOOTING
───────────────────────────────────────────────────────────────

  "ibkr-options MCP not connected"
    → TWS must be running and logged in
    → Check API settings: TWS → Edit → Global Configuration →
      API → Settings → "Enable ActiveX and Socket Clients"
    → Verify port matches server.py (7497 paper / 7496 live)

  "Tool count shows 17 not 18"
    → server.py is still v2.2 — replace with v2.3
    → Quit Claude Desktop fully and reopen

  "scan times out"
    → Normal for first run after market open (data caching)
    → Run during market hours for fastest response
    → If still timing out, restart TWS

  "get_open_positions returns nothing but I have positions"
    → Confirm TWS is connected to right account
    → Paper vs Live accounts have separate positions
    → Verify port number matches the account

  "Audit fails on a setup that looks good"
    → Trust the audit. It exists to save you from yourself.
    → Document the rejection in your notes for monthly review
    → If a SPECIFIC rule keeps blocking GOOD trades after 20+
      examples, that's data for tightening or loosening it.
      Don't override on a hunch.
